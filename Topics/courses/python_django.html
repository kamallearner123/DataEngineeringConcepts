20-Hour Django & Python Web Development Course

Build Web Applications with Python's Powerful Framework Django

Course Overview

This intensive 20-hour course is designed for individuals with basic Python knowledge who want to dive into web development using the Django framework. You'll learn core Django concepts, build practical projects, and gain a solid understanding of full-stack web development.

Duration: 20 hours (10 chapters, approx. 2 hours each)

Level: Beginner to Intermediate (Basic Python knowledge assumed)

Projects: 3 hands-on projects

Focus: Full-stack web development with Django, Python, HTML, CSS, JavaScript basics

Course Curriculum

Chapter 1: Python & Web Fundamentals Refresher (2 Hours)

Objective: Solidify Python basics and understand web development concepts relevant to Django.

  • Python refresh: Functions, classes, modules, virtual environments (`venv`, `pip`).
  • Web fundamentals: HTTP/HTTPS, client-server model, HTML, CSS, JavaScript (brief overview).
  • Introduction to Web Frameworks: Why Django? MVT (Model-View-Template) architecture.
  • Setting up Django: Installation, creating a project and your first app.
  • Running the Django development server.

Chapter 2: Django Models - Data Representation (2 Hours)

Objective: Design database schemas using Django's ORM.

  • Introduction to ORM (Object-Relational Mapper) and SQLite.
  • Defining Models: `models.Model`, common field types (CharField, IntegerField, DateTimeField, ForeignKey).
  • Model relationships: One-to-Many, Many-to-Many, One-to-One.
  • Migrations: `makemigrations` and `migrate` for database schema management.
  • Interacting with the database via Django Shell (CRUD operations).

Chapter 3: Django Views - Logic & Routing (2 Hours)

Objective: Process requests and define application logic.

  • Understanding Views: Function-based views (`HttpRequest`, `HttpResponse`).
  • URL Routing (`urls.py`): Defining URL patterns with `path()`.
  • Dynamic URLs, path converters, and named URLs.
  • Including URLs from apps into the project's `urls.py`.
  • Passing data from views to templates using `render()`.

Chapter 4: Django Templates - Presentation (2 Hours)

Objective: Create dynamic HTML pages using Django's templating engine.

  • Introduction to Django Template Language (DTL).
  • Displaying variables (`{{ }}`), using template tags (`{% %}`), and filters.
  • Control flow in templates: Loops (`for`), conditionals (`if`).
  • Template inheritance: `{% extends %}`, `{% block %}` for reusable layouts.
  • Static files management (CSS, JavaScript, images): `STATIC_URL`, `STATICFILES_DIRS`.

Project 1: Simple Blog Application (3 Hours)

Goal: Build a basic blog to list posts and view individual post details.

  • Models: `Post` (title, content, publish_date, author - ForeignKey to User).
  • Views: Function-based views to display a list of all posts and a single post.
  • URLs: Set up routes for home page, post list, and individual post details.
  • Templates: Create `base.html` for common layout, `post_list.html`, `post_detail.html`.
  • Static Files: Include basic CSS for styling.

Chapter 5: Django Forms - Handling User Input (2 Hours)

Objective: Create and process web forms for user interaction.

  • Introduction to Django Forms API.
  • Creating forms from scratch (`forms.Form`): Defining fields and custom validation.
  • Model Forms (`forms.ModelForm`): Automatically generating forms from models.
  • Handling form submissions in views (GET vs. POST requests).
  • Displaying form errors and ensuring CSRF protection.

Chapter 6: Django Admin Interface & User Authentication (2 Hours)

Objective: Leverage Django's powerful built-in features for administration and user management.

  • Introduction to Django Admin: Registering models, customizing display.
  • Creating superusers and managing data through the admin panel.
  • Django's built-in Authentication system: Users, Groups, Permissions.
  • Implementing user registration, login, and logout functionalities.
  • Restricting access with `login_required` decorator.

Project 2: Interactive To-Do List (2 Hours)

Goal: Build a simple To-Do List application with CRUD operations and user authentication.

  • Models: `TodoItem` (title, description, completed, user - ForeignKey).
  • Forms: Create a `TodoForm` for adding/editing items.
  • Views: List, add, update, delete To-Do items. Ensure only logged-in users can manage their own tasks.
  • Templates: Forms for input, list display.
  • Admin: Integrate To-Do items into Django Admin.

Chapter 7: Class-Based Views (CBVs) & Mixins (1.5 Hours)

Objective: Streamline code and leverage Django's powerful CBV system.

  • Introduction to CBVs: Advantages and common types (`TemplateView`, `ListView`, `DetailView`).
  • Using `CreateView`, `UpdateView`, `DeleteView` for CRUD operations.
  • Understanding mixins: `LoginRequiredMixin`, `UserPassesTestMixin`.
  • Refactoring function-based views to CBVs.

Chapter 8: Media Files & Messages Framework (1.5 Hours)

Objective: Handle user-uploaded files and provide user feedback.

  • Configuring Media files: `MEDIA_ROOT`, `MEDIA_URL`.
  • Handling file uploads with `ImageField` and `FileField`.
  • Serving media files in development.
  • The Django Messages Framework: Displaying one-time notifications (success, error, info).

Chapter 9: Advanced Topics & Deployment Concepts (1 Hour)

Objective: Explore more advanced concepts and prepare for production deployment.

  • Custom User Models (brief overview).
  • Signals in Django.
  • Overview of Django REST Framework (DRF) for APIs (what it is and why use it).
  • Deployment considerations: `DEBUG=False`, `ALLOWED_HOSTS`, WSGI (Gunicorn), web servers (Nginx).
  • Brief mention of database choices for production (PostgreSQL).

Chapter 10: Testing & Continuous Learning (1 Hour)

Objective: Understand the importance of testing and identify resources for ongoing growth.

  • Introduction to testing in Django: Unit tests, integration tests.
  • Writing simple tests for models and views.
  • Debugging techniques in Django (PDB, VS Code debugger).
  • Performance optimization tips.
  • Resources for continuous learning: Django documentation, community, advanced topics.

Project 3: Simple Polling/Voting App (2 Hours)

Goal: Create a web application where users can create polls and vote on choices.

  • Models: `Question` (question_text, pub_date), `Choice` (choice_text, votes).
  • Forms: For creating new questions and choices.
  • Views: Display poll list, detail view for voting, results view.
  • Authentication: Ensure only logged-in users can create polls.
  • CBVs: Utilize `ListView`, `DetailView`, `CreateView` for efficiency.
  • Messages: Provide feedback on successful votes or errors.

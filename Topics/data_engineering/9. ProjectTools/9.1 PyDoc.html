Guide to Pydoc: Python's Documentation Generator

Pydoc is a built-in Python module that generates documentation from Python modules, classes, functions, and methods. It extracts docstrings and other information to create text or HTML documentation. This guide explains Pydoc's usage with practical examples.

What is Pydoc?

Pydoc is part of Python's standard library and serves two main purposes:

  • Command-line tool: Generate documentation via the terminal.
  • Module: Import and use Pydoc programmatically in Python scripts.

It automatically extracts docstrings and metadata (e.g., function signatures, class attributes) to produce documentation in plain text, HTML, or serve it via a web server.

Using Pydoc from the Command Line

Pydoc can be run from the command line using the python -m pydoc command. Below are common use cases with examples.

Example 1: Viewing Documentation for a Module

To view documentation for a built-in module like os, run:

python -m pydoc os

Output: Displays the docstring and details of the os module in plain text in the terminal, including functions, classes, and their descriptions.

Example 2: Generating HTML Documentation

To generate an HTML file for a module, use the -w flag:

python -m pydoc -w os

Output: Creates an os.html file in the current directory with formatted documentation.

Example 3: Starting a Local Web Server

Pydoc can serve documentation via a local web server:

python -m pydoc -b

Output: Opens a browser window with a web interface to browse Python module documentation, typically at http://localhost:8000.

Example 4: Viewing Documentation for a Specific Function

To view documentation for a specific function, use dot notation:

python -m pydoc os.path.join

Output: Shows the docstring for the os.path.join function, describing its purpose and parameters.

Using Pydoc Programmatically

Pydoc can be imported as a module to generate documentation within Python scripts. Below are examples of programmatic usage.

Example 5: Generating Documentation for a Custom Module

Suppose you have a Python file named my_module.py:

# my_module.py
"""This is a sample module for Pydoc demonstration."""

def greet(name):
    """Return a greeting message for the given name."""
    return f"Hello, {name}!"

class MyClass:
    """A simple class for demonstration."""
    def __init__(self, value):
        """Initialize with a value."""
        self.value = value

    def display(self):
        """Display the stored value."""
        return self.value

Run the following command to generate documentation:

python -m pydoc ./my_module

Output: Displays the module's docstring, function, and class details in the terminal.

Example 6: Programmatic Documentation with Pydoc

You can use Pydoc in a script to generate documentation programmatically:

# generate_doc.py
import pydoc
import my_module

# Generate plain text documentation
doc = pydoc.plain(pydoc.render_doc(my_module))
print(doc)

Output: Prints the documentation for my_module in plain text format.

Example 7: Generating HTML Documentation Programmatically

To generate HTML documentation in a script:

# generate_html_doc.py
import pydoc
import my_module

# Generate HTML documentation and save to a file
html_doc = pydoc.HTMLDoc().document(my_module)
with open("my_module.html", "w") as f:
    f.write(html_doc)

Output: Creates my_module.html with formatted HTML documentation for my_module.

Key Features of Pydoc

  • Docstring Extraction: Automatically extracts and formats docstrings from modules, classes, and functions.
  • Multiple Output Formats: Supports plain text, HTML, and web server output.
  • Search Functionality: Use python -m pydoc -k to search for modules containing a keyword.
  • Interactive Help: Access Pydoc via Python's help() function, e.g., help(os).

Example 8: Searching for Modules

To search for modules related to "http":

python -m pydoc -k http

Output: Lists modules like http.client, http.server, etc.

Tips for Effective Pydoc Usage

  • Write clear and descriptive docstrings in your code to improve Pydoc output.
  • Use the -w flag to create shareable HTML documentation.
  • Use the web server (-b) for an interactive documentation experience.
  • Combine Pydoc with tools like Sphinx for more advanced documentation needs.

Limitations

  • Pydoc relies on docstrings; poorly documented code results in limited output.
  • It lacks advanced formatting options compared to tools like Sphinx.
  • It does not support custom templates for HTML output.

This guide covers the core functionality of Pydoc with practical examples. For more details, run python -m pydoc pydoc to explore Pydoc's own documentation!

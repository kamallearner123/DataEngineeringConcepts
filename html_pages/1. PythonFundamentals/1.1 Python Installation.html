Python Installation & Setup

Installing Python on Windows

To install Python on Windows, visit the official Python website at python.org/downloads. Download the latest version (e.g., Python 3.11). Run the installer and ensure you check the box for Add Python to PATH before clicking "Install Now".

Verify the installation by opening Command Prompt and running:

python --version

This should display the installed Python version (e.g., Python 3.11.4).

Installing Python on macOS

Download the macOS installer from python.org/downloads. Run the installer and follow the prompts. Alternatively, use Homebrew for a more streamlined installation:

brew install python

Verify the installation by opening Terminal and running:

python3 --version

Note: Use python3 to avoid conflicts with the system Python (Python 2).

Installing Python on Linux

Most Linux distributions come with Python pre-installed. To install or upgrade Python, use your package manager. For Ubuntu/Debian, run:

sudo apt update
sudo apt install python3

For Fedora, use:

sudo dnf install python3

Verify the installation:

python3 --version

Setting Up a Python Development Environment

A good Integrated Development Environment (IDE) enhances Python development. Popular choices include:

  • Visual Studio Code (VS Code): Lightweight, with Python extensions for linting, debugging, and IntelliSense. Download from code.visualstudio.com.
  • PyCharm: Feature-rich IDE for professional developers, available at jetbrains.com/pycharm.

For VS Code, install the Python extension and configure it to use your Python interpreter:

python3 -m pip install --user pylint

Using Python Virtual Environments

Virtual environments isolate project dependencies. Create a virtual environment using:

python3 -m venv myenv

Activate it:

  • Windows:
    • myenv\Scripts\activate
  • macOS/Linux:
    • source myenv/bin/activate

Deactivate by running:

deactivate

List installed packages:

Upgrade a package:

Use requirements.txt to manage dependencies:

# requirements.txt
requests==2.28.1
numpy==1.24.3

Install from requirements.txt:

pip install -r requirements.txt

Challenging Questions

Question 1: What happens if you forget to check "Add Python to PATH" during Windows installation, and how can you fix it?

Show Answer

Answer: If "Add Python to PATH" is not checked, running python in Command Prompt will result in a "command not found" error.

Fix: Manually add Python to the PATH environment variable or reinstall Python with the option enabled. To add manually, edit the PATH variable in System Properties to include the Python installation directory (e.g., C:\Python311) and Scripts directory.

Question 2: Why might pip install a package globally instead of in a virtual environment, and how can you ensure it installs in the virtual environment?

Show Answer

Answer: If the virtual environment is not activated, pip installs packages globally.

Fix: Activate the virtual environment before running pip install (e.g., source myenv/bin/activate on macOS/Linux). Verify the active environment by checking that the terminal prompt shows the environment name.

Question 3: What is the output of the following command in a virtual environment, and why?

pip list

Assume the virtual environment was just created and no additional packages were installed.

Show Answer

Answer: The output will show only pip and setuptools with their versions (e.g., pip 23.2.1, setuptools 68.0.0).

Explanation: A newly created virtual environment includes only pip and setuptools by default, as these are essential for package management. No other packages are installed until explicitly added.

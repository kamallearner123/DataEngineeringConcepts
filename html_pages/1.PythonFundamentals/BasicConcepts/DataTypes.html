Python Variables and Data Types

Explore the basics of Python programming with variables and essential data types!

Variables

Variables in Python are like containers that store data. You don't need to declare the type of a variable explicitly; Python figures it out dynamically.

Interesting Point: Python variables are references to objects, not fixed memory locations. This means reassigning a variable can point it to a completely different type of data! 
x = 10  # x points to an integer
x = "Hello"  # Now x points to a string!

Integer (int)

Integers are whole numbers without decimal points. Python integers have unlimited precision, so you can work with very large numbers.

Interesting Point: Python automatically switches to a "bignum" representation for large integers, so you can calculate something like 2**1000 without overflow! 
big_number = 2 ** 1000
print(big_number)  # Outputs a massive number!

Float

Floats represent numbers with decimal points. They are useful for precise calculations but can have rounding errors due to how computers store them.

Interesting Point: Floating-point arithmetic can lead to surprises, like 0.1 + 0.2 != 0.3 due to binary representation limitations. 
print(0.1 + 0.2)  # Outputs 0.30000000000000004
Challenge: Write code to add 0.1 and 0.2. Check if the result equals 0.3 using ==. How can you handle this precision issue?

String (str)

Strings are sequences of characters enclosed in quotes. They are immutable, meaning you can't change a string directly.

Interesting Point: Strings support powerful methods like .join() and .format(), and f-strings make formatting super easy! 
name = "Alice"
greeting = f"Hello, {name}!"
print(greeting)  # Outputs: Hello, Alice!

List

Lists are ordered, mutable collections of items. They can hold mixed data types and are great for storing sequences.

Interesting Point: Lists are mutable, but assigning a list to another variable creates a reference, not a copy. Changes to one affect the other! 
list1 = [1, 2, 3]
list2 = list1
list2.append(4)
print(list1)  # Outputs: [1, 2, 3, 4]
Challenge: Create a list of 3 fruits. Make a copy of the list and add a new fruit to the copy. Check if the original list changed.

Tuple

Tuples are ordered, immutable collections. They are faster than lists and often used for fixed data.

Interesting Point: Tuples can be used as dictionary keys because they are immutable and hashable, unlike lists. 
coords = (10, 20)
print(coords[0])  # Outputs: 10
Challenge: Create a tuple with 3 numbers. Try to change one of them. What error do you get? Why?

Dictionary (dict)

Dictionaries store key-value pairs. Keys must be immutable and unique, while values can be any type.

Interesting Point: Since Python 3.7, dictionaries maintain insertion order, making them even more versatile! 
person = {"name": "Bob", "age": 25}
print(person["name"])  # Outputs: Bob
Challenge: Create a dictionary with 3 key-value pairs about yourself. Add a new key-value pair and print the dictionary. Does the order matter?

Set

Sets are unordered collections of unique items. They are great for removing duplicates and performing set operations like union and intersection.

Interesting Point: Sets are highly optimized for membership testing, making in operations very fast! 
nums = {1, 2, 2, 3}
print(nums)  # Outputs: {1, 2, 3}
